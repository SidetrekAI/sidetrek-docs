# Meltano CLI

If you want to use meltano directly using CLI, instead of dagster UI, you can do so, by running:

<BashCode code="cd <example-project>/<example-project>/meltano" client:load />
<BashCode code="sidetrek run meltano --version" client:load />


It will show you the version of meltano installed.

## Installing Plugins

You can install a plugin running this command:

<BashCode code="sidetrek run meltano add <plugin-type> <plugin-name>" client:load />
For example:
<BashCode code="sidetrek run meltano add extractor tap-gitlab" client:load />
or
<BashCode code="sidetrek run meltano add loader target-postgres" client:load />
 

If multiple variants of a discoverable plugin are available, you can choose a specific (non-default) variant using the `--variant` option on `meltano add`:

<BashCode code="sidetrek run meltano add <type> <name> --variant <variant>" client:load />
For example:
<BashCode code="sidetrek run meltano add loader target-postgres --variant=transferwise" client:load />

## Installing Custom Plugins

Custom plugins for packages can be added to your project using the `--custom`:

<BashCode code="sidetrek run meltano add --custom <type> <name>" client:load />
For example:
<BashCode code="sidetrek run meltano add --custom loader target-iceberg" client:load />

Add value for `namespace` , `pip_url`, etc.

## Configuring Plugins

To set the configuration to the newly installed plugin, run this:

<BashCode code="sidetrek run meltano config <name> set <setting> <value>" client:load />
For example:
<BashCode code="sidetrek run meltano config target-postgres set password my_password" client:load />

You can use `--interactive` flag to set the configuration settings using CLI:
<BashCode code="sidetrek run meltano config <name> set --interactive" client:load />


Running this will show you a list of all configurations.

However, you can also, add configuration to `meltano.yml` file:

```yaml title="example-project>/meltano/meltano.yml"
...
plugins:
  extractors:
    - name: tap-csv
      variant: meltanolabs
      pip_url: git+https://github.com/MeltanoLabs/tap-csv.git
      config:
        csv_files_definition: extract/example_csv_files_def.json
  loaders:
  ...
```

You need to install the changes, using 
<BashCode code="meltano install <plugin-type> <plugin-name>" client:load />

If you change any configuration in the `meltano.yml` file, such as `pip-url` you can run `meltano install --clean <plugin-type> <plugin-name>`

## Explicit inheritance

If you want to use the same plugin with different configuration, you can do so using explicit inheritance. 

<BashCode code="sidetrek run meltano add <type> <name> --inherit-from <discoverable-name>" client:load />
Or equivalently:
<BashCode code="sidetrek run meltano add <type> <discoverable-name> --as <name>" client:load />
For example:
<BashCode code="sidetrek run meltano add extractor tap-postgres--billing --inherit-from tap-postgres" client:load />
or
<BashCode code="sidetrek run meltano add extractor tap-postgres --as tap-postgres--billing" client:load />

The corresponding inheriting plugin definition in your `meltano.yml` project file  will use `inherit_from`:

```yaml title="example-project>/meltano/meltano.yml"
...
plugins:
  extractors:
  - name: tap-postgres--billing
    inherit_from: tap-postgres
    variant: transferwise
    pip_url: pipelinewise-tap-postgres
...
```

## Updating plugins

You can update a plugin in your project using the `--update` option. Updating a plugin will re-add it to your project - that is:

- Update the plugin lock file the same as `meltano lock --update` would
- Update the plugin entry in the `meltano.yml`, without overwriting any user-defined config or extras

<BashCode code="sidetrek run meltano add --update <type> <name>" client:load />
For example:
<BashCode code="sidetrek run meltano add --update extractor tap-gitlab" client:load />
