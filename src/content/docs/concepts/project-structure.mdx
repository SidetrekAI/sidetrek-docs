# Sidetrek Project Structure
The project is organized into several directories and files, each serving a specific purpose. Below is an overview of the structure:

```plaintext
demo_project
├── .env
├── .gitignore
├── .sidetrek
├── README.md
├── docker-compose.yaml
├── demo_project
│   ├── __init__.py
│   ├── dagster
│   ├── data
│   ├── dbt
│   └── meltano
├── poetry.lock
├── pyproject.toml
├── sidetrek.config.yaml
├── superset
└── trino
└── etc
```

### Root Directory

- **.env**: Environment variables for the project.
- **.gitignore**: Specifies files and directories to be ignored by Git.
- **.sidetrek**: Configuration for the Sidetrek tool.
- **README.md**: Project overview and documentation.
- **docker-compose.yaml**: Configuration for Docker Compose to manage multi-container Docker applications.
- **poetry.lock**: Dependency lock file for Poetry.
- **pyproject.toml**: Project configuration for Poetry and other tools.
- **sidetrek.config.yaml**: Configuration for Sidetrek.

### demo_project Directory

- **init.py**: Initializes the `demo_project` Python package.
- **dagster/**: Contains Dagster configurations and files.
- **data/**: Directory for data files, including an `example` subdirectory.
- **dbt/**: DBT (Data Build Tool) configurations and logs.
- **meltano/**: Meltano project directory with its configurations, plugins, and requirements.

### superset Directory

Contains the source code and configuration files for Apache Superset, a data exploration and visualization platform.

### trino Directory

Configuration files for Trino (formerly PrestoSQL), a distributed SQL query engine.

### Why is Trino and Superset Outside the Project Folder?

Trino and Superset are not included within the second project folder because they are deployed as separate services rather than being part of the user code. These tools are independently managed and deployed, ensuring that any changes made to the user code do not affect their deployment. This separation allows for independent updates and maintenance of Trino and Superset without impacting the core project, thereby facilitating a more modular and manageable deployment architecture.

### dagster Directory

Contains Dagster-related configurations and code.

```plaintext
demo_project
...
└── demo_project
    ...
    └── dagster
        └── demo_project
            ├── .env
            ├── .gitignore
            ├── .logs_queue
            ├── .nux
            ├── .telemetry
            │   └── id.yaml
            ├── README.md
            ├── demo_project
            │   ├── __init__.py
            │   ├── __pycache__
            │   ├── assets.py
            │   ├── dbt_assets.py
            │   └── meltano.py
            ├── demo_project_tests
            │   ├── __init__.py
            │   └── test_assets.py
            ├── history
            │   ├── runs
            │   └── runs.db
            ├── logs
            │   └── event.log
            ├── pyproject.toml
            ├── schedules
            │   └── schedules.db
            ├── setup.cfg
            ├── setup.py
            └── storage
                ├── 8407b3f3-504d-4a32-91c2-b447a485dc5c
                └── abe21850-81be-4de9-932a-017194e6ff35
```

- **.env**: Environment variables specific to Dagster.
- **.gitignore**: Files to be ignored by Git.
- **.logs_queue, .nux, .telemetry**: Dagster-specific metadata and configurations.
- **README.md**: Documentation for the Dagster project.
- **demo_project/**: Core Dagster project files.
    - **init.py**: Initializes the `demo_project` module.
    - **assets.py, dbt_assets.py, meltano.py**: Python scripts defining various assets and integrations.
- **demo_project_tests/**: Contains tests for the Dagster project.
    - **init.py**: Initializes the `demo_project_tests` module.
    - **test_assets.py**: Test cases for the assets.
- **history/**: Stores historical data for runs.
    - **runs/**: Directory for individual run data.
    - **runs.db**: SQLite database file for storing run history.
- **logs/**: Directory for logs.
    - **event.log**: Log file for events.
- **pyproject.toml, setup.cfg, setup.py**: Configuration files for building and packaging the project.
- **schedules/**: Directory for schedule data.
    - **schedules.db**: SQLite database for schedules.
- **storage/**: Directory for storage files.

### data Directory

Contains data files for the project.

```plaintext
demo_project
...
└── demo_project
    ...
    └── data
        └── example
            ├── customers.csv
            ├── orders.csv
            ├── products.csv
            └── stores.csv
```

- **example/**: Example data files.
    - **customers.csv, orders.csv, products.csv, stores.csv**: CSV files containing example data.

        │   ├── target

- **demo_project/**: Core DBT project files.
    - **.gitignore, .user.yml, README.md**: Configuration and documentation files.

    - **dbt_packages/**: Directory for DBT packages.
    - **dbt_project.yml**: DBT project configuration.
    - **logs/**: Log files for DBT operations.
        - **dbt.log**: Log file for DBT.
    - **macros/**: Directory for DBT macros (contains a placeholder `.gitkeep`).
    - **models/**: Directory for DBT models.
        - **intermediate, marts, staging**: Subdirectories for different types of models.
    - **profiles.yml**: DBT profiles configuration.
    - **seeds/**: Directory for seed data (contains a placeholder `.gitkeep`).
    - **snapshots/**: Directory for snapshots (contains a placeholder `.gitkeep`).
    - **target/**: Directory for target files.
        - **compiled, dbt.log, graph.gpickle, manifest.json, etc.**: Various files generated by DBT during runs.
    - **tests/**: Directory for tests (contains a placeholder `.gitkeep`).
- **logs/**: Directory for log files.
    - **dbt.log**: Log file for DBT.

### meltano Directory

Contains Meltano project files and configurations.

```plaintext
...
└── demo_project
    ...
    └── meltano
        ├── .env
        ├── .gitignore
        ├── .meltano
        │   ├── analytics.json
        │   ├── cache
        │   │   └── discovery.yml
        │   ├── extractors
        │   │   └── tap-csv
        │   ├── loaders
        │   │   └── target-iceberg
        │   ├── logs
        │   │   ├── elt
        │   │   └── pip
        │   ├── meltano.db
        │   └── run
        │       ├── bin
        │       └── meltano.yml.lock
        ├── README.md
        ├── analyze
        │   └── .gitkeep
        ├── extract
        │   ├── .gitkeep
        │   └── example_csv_files_def.json
        ├── load
        │   └── .gitkeep
        ├── meltano.yml
        ├── notebook
        │   └── .gitkeep
        ├── orchestrate
        │   └── .gitkeep
        ├── output
        │   └── .gitignore
        ├── plugins
        │   └── extractors
        │       └── tap-csv--meltanolabs.lock
        ├── requirements.txt
        └── transform
            └── .gitkeep
```

- **.env, .gitignore**: Environment variables and Git ignore file for Meltano.
- **.meltano/**: Core Meltano configuration and state.
    - **analytics.json, discovery.yml, meltano.db, etc.**: Various configuration and state files.
- **README.md**: Documentation for the Meltano project.
- **analyze/, extract/, load/, notebook/, orchestrate/, output/**: Directories for different stages of ELT (Extract, Load, Transform) and analysis.
    - extract folder contains `example_csv_files_def.json` which is used in `meltano.yml` for defining files for the extractor `tap-csv`
    - Contains `.gitkeep` or other relevant files.
- **meltano.yml**: Main configuration file for Meltano.
- **plugins/**: Directory for plugins.
    - **extractors/**: Contains extractor plugins.
- **requirements.txt**: Python dependencies for Meltano.
- **transform/**: Directory for transformation files (contains a placeholder `.gitkeep`).
